# Система генерации B2B-лидов для grifun.ru

## Обзор

Сайт преобразован в систему регулярной генерации лидов с фокусом на конверсию намерения в контакт.

## Реализованные компоненты

### 1. Лид-магнит (AI-аудит)

**Компонент:** `components/LeadMagnetForm.tsx`

**Особенности:**
- Многошаговая форма (3 шага)
- Сбор информации о бизнесе:
  - Контакты (имя, email/телефон, компания)
  - Ниша бизнеса
  - Размер команды
  - Где больше всего ручной работы
  - Главная цель автоматизации
  - Детали задачи

**Результат:**
- Данные отправляются в Planfix CRM
- Автоматический редирект на страницу благодарности

### 2. Страница благодарности

**Компонент:** `app/thank-you/page.tsx`

**Содержание:**
- Чёткое ожидание (ответ в течение 2 часов)
- Пошаговый план дальнейших действий
- Дополнительные CTA (кейсы, документы)
- Сообщения о доверии (без спама, конфиденциально)

### 3. Микро-CTA по всему сайту

**Обновлённые компоненты:**
- `Services.tsx`: "Узнать, подойдёт ли это моему бизнесу"
- `ValueProposition.tsx`: "Посчитать окупаемость для моего бизнеса" + "Получить план автоматизации"
- `TypicalResults.tsx`: "Узнать, что можно автоматизировать за 14 дней"
- `Cases.tsx`: "Получить такой же результат" (основной CTA) + "Подробнее о проекте"

**Принцип:** Каждый смысловой блок заканчивается конкретным действием, а не абстрактным "Связаться".

### 4. Блоки доверия

**Компонент:** `components/TrustBlock.tsx`

**Содержание:**
- "Почему нам доверяют": 4 пункта (честность, фокус на результат, прозрачность, безопасность)
- "Как мы работаем": 4 этапа (аудит, прототип, разработка, запуск)

### 5. Страницы под интенты

**Создано:**
- `/services/ai-bot` — AI-бот для бизнеса

**Структура каждой страницы:**
- Проблема (что не так сейчас)
- Решение (что мы предлагаем)
- Кейс (конкретный результат с цифрами)
- CTA (форма контакта)

**Компонент:** `components/FAQ.tsx`

**Вопросы:**

## Метрики для отслеживания

### Ключевые события (требуется настройка аналитики):

1. **Форма AI-аудита:**
   - `lead_magnet_start` — начало заполнения формы
   - `lead_magnet_step_1` — завершение шага 1 (контакты)
   - `lead_magnet_step_2` — завершение шага 2 (о бизнесе)
   - `lead_magnet_submit` — отправка формы
   - `lead_magnet_success` — успешная отправка

2. **Микро-CTA:**
   - `cta_click_services` — клик на CTA в блоке услуг
   - `cta_click_value_prop` — клик на CTA в блоке преимуществ
   - `cta_click_cases` — клик на CTA в кейсах
   - `cta_click_typical_results` — клик на CTA в типовых результатах

3. **Навигация:**
   - `scroll_to_contact` — прокрутка до формы контакта
   - `faq_expand` — раскрытие вопроса в FAQ

### Целевые метрики:

- **Конверсия посетитель → лид:** Цель — 3-5%
- **Конверсия лид → диалог:** Цель — 60-80% (через редирект на thank-you)
- **Стоимость лида:** Отслеживать через UTM-метки в рекламе

## Технические детали

### Интеграции:

- **Planfix CRM:** Все заявки автоматически создаются как задачи
- **Файловое хранение:** Резервное копирование всех лидов в `data/leads/`

### Форма контакта:

- Поддержка двух типов заявок: "консультация" и "обратный звонок"
- Автоподстановка услуги из query-параметра `?service=ai-audit`
- Редирект на `/thank-you?type=consultation` после успешной отправки

### SEO:

- Правильная структура заголовков (H1, H2, H3)
- FAQ-блок для длинных запросов
- Мета-теги для каждой страницы
- Canonical URLs
